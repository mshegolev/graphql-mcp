2. **`Dockerfile`** — Multi-stage build. This project uses maturin (Rust native extension), so the build stage needs both Rust and Python.

   ```dockerfile
   # ---- Build stage ----
   FROM python:3.12-slim AS builder

   # Install Rust toolchain + build deps for maturin/pyo3
   RUN apt-get update && \
       apt-get install -y --no-install-recommends curl build-essential && \
       curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y && \
       rm -rf /var/lib/apt/lists/*

   ENV PATH="/root/.cargo/bin:$PATH"

   WORKDIR /build

   # Copy source
   COPY pyproject.toml ./
   COPY native/ native/
   COPY src/ src/

   # Build wheel with maturin (includes Rust extension)
   RUN pip install --no-cache-dir maturin && \
       maturin build --release --strip -o dist/

   # ---- Runtime stage ----
   FROM python:3.12-slim AS runtime

   # Create non-root user
   RUN groupadd --gid 1000 appuser && \
       useradd --uid 1000 --gid 1000 --shell /bin/bash appuser

   WORKDIR /app

   # Install the built wheel with all optional deps
   COPY --from=builder /build/dist/*.whl /tmp/
   RUN pip install --no-cache-dir /tmp/*.whl[all] && \
       rm -rf /tmp/*.whl

   # Switch to non-root
   USER appuser

   EXPOSE 8000

   # Health check using /health endpoint
   HEALTHCHECK --interval=30s --timeout=5s --start-period=10s --retries=3 \
       CMD python -c "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')" || exit 1

   CMD ["graphql-mcp", "serve"]
   ```

   Key decisions:
   - **python:3.12-slim** base — matches project's Python target, slim for size.
   - **Rust in build stage only** — not carried to runtime (multi-stage).
   - **`pip install *.whl[all]`** — installs server + mcp + cli extras so `graphql-mcp serve` works.
   - **Non-root `appuser`** (UID 1000) — standard security practice.
   - **HEALTHCHECK** uses `/health` (always 200). Kubernetes probes would use `/health` for liveness and `/ready` for readiness (configured in k8s manifests, not Dockerfile).
   - **`CMD ["graphql-mcp", "serve"]`** — exec form, runs the CLI serve command which starts uvicorn on 0.0.0.0:8000.

   NOTE on `pip install *.whl[all]`: pip may not support extras from wheel file path directly. If that's the case, install the wheel first, then install extras separately:
   ```dockerfile
   RUN pip install --no-cache-dir /tmp/*.whl && \
       pip install --no-cache-dir 'fastapi>=0.100,<1' 'uvicorn>=0.20,<1' 'mcp>=1.0,<2' 'click>=8,<9' && \
       rm -rf /tmp/*.whl
   ```
   Or install from the wheel with extras using the exact filename. Test which approach works during execution.
