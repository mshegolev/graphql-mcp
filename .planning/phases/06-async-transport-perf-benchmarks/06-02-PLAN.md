**Create `src/graphql_mcp/adapters/inbound/async_lib.py`:**

Mirror `lib.py` structure exactly, with these changes:

1. Import `AsyncHttpTransport` instead of `HttpTransport`
2. Constructor accepts `transport: AsyncHttpTransport | None` (not `HttpTransport`)
3. `async def query(...)` — mutation guard is sync (call `check_mutation_guard` normally), then `await self._transport.execute(...)`. Returns `QueryResult`.
4. `async def raw(...)` — mutation guard on body["query"] is sync, then `await self._transport.post_raw(body)`. Returns `QueryResult`.
5. `introspect()`, `describe_type()`, `list_subgraphs()`, `refresh_schema()` are **NOT async** — they use `SchemaService.resolve()` and `SchemaAnalyzer` which are sync. Keep them as regular sync methods. This matches reality: schema is resolved from cache or local sources, not through the async transport.
6. `async def close(self)` — `await self._transport.close()` with `_closed` idempotency flag.
7. `async def __aenter__(self) -> AsyncGraphQLClient` returns `self`
8. `async def __aexit__(self, ...)` calls `await self.close()`
9. `@classmethod def from_env(cls, **overrides)` — same cascade logic as sync, but creates `AsyncHttpTransport` instead of `HttpTransport`. Registers `atexit` but note: atexit can't await, so register a sync wrapper that calls `transport.close()` directly (httpx.AsyncClient has a sync `close()` method for cleanup). Use a lambda: `atexit.register(lambda: instance._transport and instance._transport._client.close() if not instance._closed else None)` — OR simpler: just mark `_closed` and let GC handle it. Actually, the cleanest approach: define a sync `_sync_close` helper that sets `_closed=True` and calls `self._transport._client.close()` (httpx.AsyncClient.close() is available as sync). Register that with atexit.

**IMPORTANT design note on atexit:** `atexit` handlers cannot be async. The sync `GraphQLClient` calls `self._transport.close()` which calls `httpx.Client.close()` — sync. For `AsyncGraphQLClient`, the transport's close is `async`. However, `httpx.AsyncClient` does have a sync-compatible shutdown via `__del__`. The pragmatic approach: in `from_env()`, register `atexit.register(instance._sync_cleanup)` where `_sync_cleanup` is a sync method that sets `_closed=True` and calls `self._transport._client.close()` directly (accessing the underlying `httpx.AsyncClient.close()` method, which is sync-safe for cleanup). This matches httpx's documented cleanup pattern.
