Create `.github/workflows/ci.yml` with three jobs:

**Job 1: `lint-and-test`** — runs on `ubuntu-latest`:
1. Checkout code
2. Set up Python 3.11
3. Install dev + all optional deps: `pip install -e ".[dev,all]"`
4. Run `ruff check src/ tests/`
5. Run `pytest tests/ -v --tb=short`

**Job 2: `wheels`** — runs on matrix `[ubuntu-latest, macos-latest, windows-latest]`, needs `lint-and-test`:
1. Checkout code
2. Set up Python 3.11
3. Install Rust toolchain (via `dtolnay/rust-toolchain@stable`)
4. Use `pypa/cibuildwheel@v2.23` action
5. Configure CIBW environment variables:
   - `CIBW_BUILD`: `cp310-* cp311-* cp312-*` (Python 3.10, 3.11, 3.12)
   - `CIBW_SKIP`: `*-win32 *-manylinux_i686 *-musllinux_i686` (no 32-bit)
   - `CIBW_ARCHS_LINUX`: `x86_64 aarch64`
   - `CIBW_ARCHS_MACOS`: `arm64 x86_64`
   - `CIBW_ARCHS_WINDOWS`: `AMD64`
   - `CIBW_BEFORE_ALL_LINUX`: `curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh -s -- -y && export PATH="$HOME/.cargo/bin:$PATH"` (install Rust in manylinux/musllinux containers)
   - `CIBW_ENVIRONMENT_LINUX`: `PATH=$HOME/.cargo/bin:$PATH`
   - `CIBW_TEST_REQUIRES`: `pytest respx`
   - `CIBW_TEST_COMMAND`: `pytest {project}/tests/ -x --ignore={project}/tests/test_codec_parity.py` (skip parity tests in CI wheel builds — Rust ext may not be available in all containers)
6. Set up QEMU for aarch64 cross-compile on Linux: `docker/setup-qemu-action@v3`
7. Upload wheel artifacts via `actions/upload-artifact@v4`

**Job 3: `sdist`** — runs on `ubuntu-latest`, needs `lint-and-test`:
1. Checkout code
2. Set up Python 3.11
3. Install `build`: `pip install build`
4. Build sdist: `python -m build --sdist`
5. Upload sdist artifact

**Job 4: `fallback-install`** — runs on `ubuntu-latest`, needs `sdist`:
1. Checkout code
2. Set up Python 3.11 (NO Rust toolchain)
3. Download sdist artifact
4. Install from sdist with `--no-build-isolation` flag and maturin fallback — OR simpler: `pip install .` with `MATURIN_PEP517_ARGS="--skip-auditwheel"` — Actually, the orjson fallback means the package works WITHOUT the Rust extension. The maturin build-backend will fail without Rust, so this test needs a different approach.
   
   IMPORTANT: The current pyproject.toml uses maturin as build-backend. For pip install without Rust to work, the build-backend must handle the case where Rust compilation fails gracefully (maturin ≥1.0 with `[tool.maturin] bindings = "pyo3"` does NOT silently fall back). 
   
   The orjson fallback is a RUNTIME fallback — the `codec_factory.get_codec()` function returns `OrjsonCodec` when the native extension is not importable. But the BUILD step still requires Rust unless we configure it otherwise.
   
   For pip install without Rust to succeed, the package needs to be installable as a pure-Python sdist. This requires EITHER:
   - Building a pure-Python wheel (no native extension) alongside the native wheels, OR
   - Configuring maturin to produce a fallback wheel without the extension
   
   The simplest approach: In CI, produce an additional pure-Python wheel by temporarily replacing the build-backend. Use `python -m build --wheel` with a modified pyproject.toml that uses `hatchling` or `setuptools` instead of `maturin`. This is complex.
   
   SIMPLER: Use the `maturin build --skip-auditwheel` approach, which produces a wheel even without a Rust extension when the Cargo manifest is not found. Actually — maturin REQUIRES Rust.
   
   SIMPLEST APPROACH for "pip install without Rust": Users install from the pre-built wheel (which embeds the native extension). For systems where no wheel is available, the fallback is: install dependencies manually + set up as a pure-Python package. The existing cibuildwheel matrix covers all major platforms.
   
   For the CI test: Download the built wheel from the `wheels` job and install it on a clean Python 3.11 — this proves the wheel installs and `from graphql_mcp import GraphQLClient` works.

So revise Job 4 to:
1. Download wheel artifact from `wheels` job (ubuntu x86_64 cp311 wheel)
2. Set up Python 3.11 (no Rust)
3. `pip install graphql_mcp-*.whl`
4. `python -c "from graphql_mcp import GraphQLClient; print('OK')"`
5. This confirms the wheel installs without Rust and the orjson fallback path works.
