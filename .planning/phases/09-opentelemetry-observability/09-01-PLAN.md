Purpose: OTEL-01 (outbound spans + traceparent), OTEL-03 (custom query metrics), OTEL-05 (env-based OTLP config) — the tracing and metrics foundation that Wave 2 builds upon.

**2. `otel_bootstrap.py` — OTEL initialization with graceful no-op:**

Create `src/graphql_mcp/adapters/outbound/otel_bootstrap.py` with:

- A module-level `_OTEL_AVAILABLE` flag: `try: from opentelemetry import trace; _OTEL_AVAILABLE = True except ImportError: _OTEL_AVAILABLE = False`.
- `init_otel() -> None`: If `_OTEL_AVAILABLE` is False, return immediately (no-op). Otherwise:
  - Read `OTEL_SERVICE_NAME` from env (default `"graphql-mcp"`).
  - Read `OTEL_EXPORTER_OTLP_ENDPOINT` from env (default `None`).
  - Create a `Resource` with `service.name`.
  - Create `TracerProvider(resource=resource)`.
  - If `OTEL_EXPORTER_OTLP_ENDPOINT` is set, add `BatchSpanProcessor(OTLPSpanExporter())` — the OTLPSpanExporter reads the endpoint from env automatically.
  - Create `MeterProvider(resource=resource, metric_readers=[...])`.
  - If `OTEL_EXPORTER_OTLP_ENDPOINT` is set, add `PeriodicExportingMetricReader(OTLPMetricExporter())`.
  - Call `trace.set_tracer_provider(provider)` and `metrics.set_meter_provider(meter_provider)`.
  - Instrument httpx: `from opentelemetry.instrumentation.httpx import HTTPXClientInstrumentor; HTTPXClientInstrumentor().instrument()`. This auto-instruments ALL httpx.Client/AsyncClient instances globally — outbound HTTP calls get spans with `http.method`, `http.url` attributes and W3C traceparent propagation. No changes needed in HttpTransport/AsyncHttpTransport constructors.
  - Instrument logging (for OTEL-04 in Plan 02, but the instrument call is cheap): `from opentelemetry.instrumentation.logging import LoggingInstrumentor; LoggingInstrumentor().instrument(set_logging_format=False)`.
- `get_tracer(name: str) -> trace.Tracer`: Returns `trace.get_tracer(name)` if OTEL available, else a no-op tracer (just call `trace.get_tracer(name)` — the API returns NoOpTracer when no SDK is configured).
- `get_meter(name: str) -> metrics.Meter`: Returns `metrics.get_meter(name)` if OTEL available, else call it anyway (returns NoOpMeter).
- `is_otel_available() -> bool`: Returns `_OTEL_AVAILABLE`.

**Important:** `init_otel()` must be idempotent — use a module-level `_initialized` flag so calling it twice is safe.

**3. HttpTransport / AsyncHttpTransport — NO CODE CHANGES NEEDED:**

The `opentelemetry-instrumentation-httpx` library's `HTTPXClientInstrumentor().instrument()` call patches `httpx.Client` and `httpx.AsyncClient` globally at the class level. This means all existing `HttpTransport` and `AsyncHttpTransport` instances automatically get spans. No constructor changes, no import changes. The existing code remains unchanged.

**1. `otel_metrics.py` — custom metric instruments (OTEL-03):**

Create `src/graphql_mcp/adapters/outbound/otel_metrics.py`:

```python
"""Custom graphql_mcp.query.* OpenTelemetry metrics."""
from __future__ import annotations

import time
from contextlib import contextmanager
from typing import TYPE_CHECKING, Generator

from graphql_mcp.adapters.outbound.otel_bootstrap import get_meter, is_otel_available

if TYPE_CHECKING:
    from graphql_mcp.domain.models import QueryResult

_meter = get_meter("graphql_mcp")

# Create instruments at module level — they are no-ops when OTEL is unavailable
_query_duration = _meter.create_histogram(
    name="graphql_mcp.query.duration",
    description="Duration of GraphQL query execution in seconds",
    unit="s",
)
_query_count = _meter.create_counter(
    name="graphql_mcp.query.count",
    description="Total number of GraphQL queries executed",
)
_query_errors = _meter.create_counter(
    name="graphql_mcp.query.errors",
    description="Total number of GraphQL query errors by error_class",
)


def record_query_metrics(result: QueryResult, duration_s: float, operation: str = "query") -> None:
    """Record metrics for a completed query operation.

    Args:
        result: The QueryResult from the transport.
        duration_s: Wall-clock duration in seconds.
        operation: The operation name (query, raw, entities).
    """
    attrs = {"operation": operation, "error_class": result.error_class.value}
    _query_count.add(1, attrs)
    _query_duration.record(duration_s, attrs)
    if result.error_class.value != "ok":
        _query_errors.add(1, attrs)
```

**2. `lib.py` — instrument `query()`, `raw()`, `entities()` with custom metrics:**

Add import at top of `lib.py`:
```python
import time
from graphql_mcp.adapters.outbound.otel_metrics import record_query_metrics
```

Wrap each of the three methods. For `query()`:
```python
def query(self, query: str, variables: dict[str, Any] | None = None) -> QueryResult:
    if not self._allow_mutations:
        check_mutation_guard(query)
    if self._transport is None:
        return QueryResult(
            errors=[{"message": "No endpoint configured"}],
            error_class=ErrorClass.TRANSPORT,
        )
    start = time.monotonic()
    result = self._transport.execute(query, variables)
    record_query_metrics(result, time.monotonic() - start, operation="query")
    return result
```

Apply the same pattern to `raw()` (operation="raw") and `entities()` (operation="entities"). In `raw()`, the timing starts after the mutation guard check but before `self._transport.post_raw(body)`. In `entities()`, timing wraps `self._transport.execute(...)`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OTLP exporter → collector | Telemetry data leaves process to external OTLP collector |
| env vars → TracerProvider config | Untrusted env vars control export destination |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | OTLP exporter | accept | Telemetry data (span names, attributes) contains operation names and URLs but no query bodies or auth tokens. Standard OTEL practice. Collector security is operator responsibility. |
| T-09-02 | Tampering | OTEL_EXPORTER_OTLP_ENDPOINT env var | accept | Env var injection would redirect telemetry, not compromise data. Operator controls env. |
| T-09-03 | Denial of Service | Metric cardinality | mitigate | error_class has only 3 values (ok/graphql/transport), operation has 3 values (query/raw/entities) — bounded cardinality. No user-controlled label values. |
</threat_model>
