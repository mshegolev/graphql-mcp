```python
"""Structured audit logging for graphql-mcp operations.

When ``GRAPHQL_AUDIT_LOG=true``, every query/raw/entities operation emits
a structured log record via Python's ``logging`` module at INFO level.

The audit record includes:
- ``caller_ip`` — client IP (from REST request, or "local" for lib face)
- ``caller_identity`` — X-User-Id header value or "anonymous"
- ``operation`` — query / raw / entities
- ``query_hash`` — SHA-256 of the query string (first 16 hex chars)
- ``error_class`` — ok / graphql / transport
- ``latency_ms`` — operation duration in milliseconds (rounded to 2 decimal places)
- ``trace_id`` — OTEL trace ID if available, else "none"

Integrates with Phase 9 OTEL: when a traced request is active, ``trace_id``
is extracted from the current span context. When OTEL ``LoggingInstrumentor``
is active, the log record also gets ``otelTraceID`` and ``otelSpanID`` fields
automatically.
"""
from __future__ import annotations

import hashlib
import logging
from typing import Any

logger = logging.getLogger("graphql_mcp.audit")


def _get_trace_id() -> str:
    """Extract trace_id from current OTEL span, or return 'none'."""
    try:
        from opentelemetry import trace
        span = trace.get_current_span()
        ctx = span.get_span_context()
        if ctx and ctx.trace_id:
            return format(ctx.trace_id, "032x")
    except ImportError:
        pass
    except Exception:
        pass
    return "none"


def _query_hash(query_str: str) -> str:
    """Return first 16 hex chars of SHA-256 hash of the query string."""
    return hashlib.sha256(query_str.encode("utf-8")).hexdigest()[:16]


def emit_audit_log(
    *,
    operation: str,
    query_str: str,
    error_class: str,
    latency_s: float,
    caller_ip: str = "local",
    caller_identity: str = "anonymous",
) -> None:
    """Emit a structured audit log record.

    This function is a no-op when audit logging is disabled — the caller
    is responsible for checking ``config.audit_log`` before calling.
    This keeps the function pure (no config dependency) and testable.
    """
    trace_id = _get_trace_id()
    record: dict[str, Any] = {
        "event": "graphql_audit",
        "caller_ip": caller_ip,
        "caller_identity": caller_identity,
        "operation": operation,
        "query_hash": _query_hash(query_str),
        "error_class": error_class,
        "latency_ms": round(latency_s * 1000, 2),
        "trace_id": trace_id,
    }
    logger.info("audit: %s", record, extra=record)
```

Key design decisions:
1. Uses Python `logging` module at INFO level — integrates with OTEL LoggingInstrumentor from Phase 9 which injects `otelTraceID`/`otelSpanID` into log records.
2. Logger name `graphql_mcp.audit` — allows targeted log filtering/routing.
3. Hash the query (not log it raw) — prevents sensitive data leakage in logs.
4. Caller checks `config.audit_log` before calling `emit_audit_log()` — keeps the function pure.
5. `extra=record` passes structured fields into the LogRecord for JSON log formatters.

**lib.py** — Add audit logging to `query()`, `raw()`, and `entities()`:

Import at top:
```python
from graphql_mcp.adapters.outbound.audit import emit_audit_log
```

In `query()`, after `record_query_metrics(...)` and before `return result`:
```python
if self._config and self._config.audit_log:
    emit_audit_log(
        operation="query",
        query_str=query,
        error_class=result.error_class.value,
        latency_s=time.monotonic() - start,
        caller_ip=getattr(self, '_audit_caller_ip', 'local'),
        caller_identity=getattr(self, '_audit_caller_identity', 'anonymous'),
    )
```

Same pattern for `raw()` (use `body.get("query", "")` as query_str) and `entities()` (use `"_entities"` as query_str).

The `_audit_caller_ip` and `_audit_caller_identity` attributes are set per-request by the REST adapter. For direct lib usage, they default to "local"/"anonymous" via getattr.

**rest.py** — In each POST endpoint handler (`graphql_query`, `graphql_raw`, `graphql_entities`), before calling `client.query()`/`client.raw()`/`client.entities()`, set the audit context:

```python
client = _get_client()
# Set audit context for this request
client._audit_caller_ip = request.client.host if request.client else "unknown"
client._audit_caller_identity = request.headers.get("x-user-id", "anonymous")
```

This is simple and thread-safe for sync FastAPI endpoints (each runs in its own thread with its own request). For a more robust approach, use contextvars — but given the sync-endpoint model, direct attribute setting works.

**async_lib.py** — Same pattern as lib.py. Add audit logging to `query()`, `raw()`, `entities()` with the same `emit_audit_log()` call. For async, use contextvars instead of instance attributes to avoid race conditions:

Actually, since AsyncGraphQLClient instances are typically not shared across requests in FastAPI (each request gets the same singleton), the same thread-unsafe pattern applies. For correctness in async, use `contextvars.ContextVar`:

```python
import contextvars

_audit_caller_ip: contextvars.ContextVar[str] = contextvars.ContextVar("audit_caller_ip", default="local")
_audit_caller_identity: contextvars.ContextVar[str] = contextvars.ContextVar("audit_caller_identity", default="anonymous")
```

But this is overengineering for now since the REST adapter uses sync endpoints. Keep it simple: use instance attributes same as sync lib.

```python
        """emit_audit_log produces a log record with all required fields."""
        with caplog.at_level(logging.INFO, logger="graphql_mcp.audit"):
            emit_audit_log(
                operation="query",
```
