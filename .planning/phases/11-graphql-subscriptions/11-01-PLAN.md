**3. ws_transport.py — upstream WebSocket transport using graphql-transport-ws:**

Create `src/graphql_mcp/adapters/outbound/ws_transport.py` implementing `UpstreamWSTransport`:

```python
class UpstreamWSTransport:
    """Connects to an upstream GraphQL WS endpoint using graphql-transport-ws sub-protocol.

    Yields QueryResult objects from upstream subscription messages.
    Manages connection lifecycle, backpressure via bounded queue, and clean shutdown.
    """

    def __init__(
        self,
        ws_endpoint: str,
        query: str,
        variables: dict[str, Any] | None = None,
        extra_headers: dict[str, str] | None = None,
        queue_size: int = 128,
        timeout: float = 30.0,
    ) -> None: ...

    async def __aiter__(self) -> AsyncIterator[QueryResult]: ...
    async def close(self) -> None: ...
    async def __aenter__(self) -> UpstreamWSTransport: ...
    async def __aexit__(self, ...) -> None: ...
```

Implementation details:
- Use `websockets.connect(ws_endpoint, subprotocols=["graphql-transport-ws"], additional_headers=extra_headers)` for the upstream connection.
- On `__aenter__` (or first iteration): send `{"type": "connection_init"}`, wait for `{"type": "connection_ack"}` with timeout.
- After ack: send `{"type": "subscribe", "id": "1", "payload": {"query": query, "variables": variables}}`.
- Internal `_reader_task`: asyncio.Task that reads messages from upstream WS and puts them into an `asyncio.Queue(maxsize=queue_size)`. For each message:
  - `type: "next"` → parse `payload` into `QueryResult(data=payload.get("data"), errors=payload.get("errors", []), error_class=...)` and put on queue.
  - `type: "error"` → create `QueryResult(error_class=ErrorClass.GRAPHQL, errors=payload)` and put on queue, then put `None` sentinel.
  - `type: "complete"` → put `None` sentinel on queue (signals end of iteration).
- `__aiter__` yields from the queue until `None` sentinel.
- `close()` cancels reader task, sends `{"type": "complete", "id": "1"}` to upstream if still connected, closes WS connection.
- Error handling: if upstream WS disconnects unexpectedly, yield a transport error `QueryResult` then stop.
- Import guard: wrap `import websockets` in try/except ImportError with a clear error message: "Install graphql-mcp[subscriptions] for subscription support".

**1. async_lib.py — add subscribe() method to AsyncGraphQLClient:**

Add to the class:
```python
async def subscribe(
    self,
    query: str,
    variables: dict[str, Any] | None = None,
    extra_headers: dict[str, str] | None = None,
) -> AsyncIterator[QueryResult]:
    """Subscribe to a GraphQL subscription and yield results.

    Connects to the upstream subscription endpoint via WebSocket using the
    graphql-transport-ws sub-protocol. Yields QueryResult objects for each
    `next` message. Iteration ends when the upstream sends `complete`.

    Usage::

        async for result in client.subscribe("subscription { events { id } }"):
            print(result.data)

    Raises ImportError if websockets is not installed (pip install graphql-mcp[subscriptions]).
    """
    from graphql_mcp.adapters.outbound.ws_transport import UpstreamWSTransport

    ws_endpoint = self._resolve_ws_endpoint()
    if not ws_endpoint:
        yield QueryResult(
            errors=[{"message": "No subscription endpoint configured"}],
            error_class=ErrorClass.TRANSPORT,
        )
        return

    merged_headers = dict(extra_headers) if extra_headers else {}
    if self._config and self._config.bearer_token:
        merged_headers.setdefault("Authorization", f"Bearer {self._config.bearer_token}")

    queue_size = self._config.subscription_queue_size if self._config else 128

    async with UpstreamWSTransport(
        ws_endpoint=ws_endpoint,
        query=query,
        variables=variables,
        extra_headers=merged_headers or None,
        queue_size=queue_size,
        timeout=float(self._config.timeout) if self._config else 30.0,
    ) as transport:
        async for result in transport:
            if self._config and self._config.audit_log:
                emit_audit_log(
                    operation="subscribe",
                    query_str=query,
                    error_class=result.error_class.value,
                    latency_s=0.0,  # per-event, not total
                    caller_ip=self._audit_caller_ip,
                    caller_identity=self._audit_caller_identity,
                )
            yield result
```

Add helper method `_resolve_ws_endpoint`:
```python
def _resolve_ws_endpoint(self) -> str:
    """Determine the WebSocket endpoint for subscriptions.

    Uses GRAPHQL_SUBSCRIPTION_ENDPOINT if set, otherwise converts
    GRAPHQL_ENDPOINT from http(s):// to ws(s)://.
    """
    if self._config and self._config.subscription_endpoint:
        return self._config.subscription_endpoint
    if self._config and self._config.endpoint:
        ep = self._config.endpoint
        if ep.startswith("https://"):
            return "wss://" + ep[len("https://"):]
        if ep.startswith("http://"):
            return "ws://" + ep[len("http://"):]
        return ep
    return ""
```

The method must be defined as `async def subscribe(self, ...) -> AsyncIterator[QueryResult]:` using `yield` to make it an async generator. The `TYPE_CHECKING` import at the top of the file already has what we need; add `AsyncIterator` to the typing imports.

```python
@app.websocket("/graphql/subscribe")
async def websocket_subscribe(ws: WebSocket) -> None:
    """WebSocket subscription proxy using graphql-transport-ws sub-protocol.

    Acts as a protocol-level proxy: accepts graphql-transport-ws from the
    downstream client, opens an upstream subscription via UpstreamWSTransport,
    and forwards next/complete/error messages.
    """
    # Only accept graphql-transport-ws sub-protocol
    subprotocol = None
    for proto in ws.scope.get("subprotocols", []):
        if proto == "graphql-transport-ws":
            subprotocol = proto
            break

    await ws.accept(subprotocol=subprotocol or "graphql-transport-ws")

    try:
        # 1. Wait for connection_init from client
        init_msg = await ws.receive_json()
        if init_msg.get("type") != "connection_init":
            await ws.send_json({"type": "error", "payload": [{"message": "Expected connection_init"}]})
            await ws.close()
            return

        await ws.send_json({"type": "connection_ack"})

        # 2. Wait for subscribe message from client
        sub_msg = await ws.receive_json()
        if sub_msg.get("type") != "subscribe":
            await ws.send_json({"type": "error", "payload": [{"message": "Expected subscribe"}]})
            await ws.close()
            return

        sub_id = sub_msg.get("id", "1")
        payload = sub_msg.get("payload", {})
        query = payload.get("query", "")
        variables = payload.get("variables")

        # 3. Build upstream connection
        from graphql_mcp.adapters.outbound.ws_transport import UpstreamWSTransport

        config = GraphQLConfig()
        ws_endpoint = config.subscription_endpoint
        if not ws_endpoint and config.endpoint:
            ep = config.endpoint
            if ep.startswith("https://"):
                ws_endpoint = "wss://" + ep[len("https://"):]
            elif ep.startswith("http://"):
                ws_endpoint = "ws://" + ep[len("http://"):]

        if not ws_endpoint:
            await ws.send_json({
                "type": "error", "id": sub_id,
                "payload": [{"message": "No subscription endpoint configured"}],
            })
            await ws.send_json({"type": "complete", "id": sub_id})
            await ws.close()
            return

        # Forward auth headers from WS connection init payload
        extra_headers: dict[str, str] = {}
        if isinstance(init_msg.get("payload"), dict):
            for k, v in init_msg["payload"].items():
                if k.lower() in _FORWARDED_HEADERS:
                    extra_headers[k] = v
        if config.bearer_token:
            extra_headers.setdefault("Authorization", f"Bearer {config.bearer_token}")

        # 4. Proxy upstream subscription to downstream client
        async with UpstreamWSTransport(
            ws_endpoint=ws_endpoint,
            query=query,
            variables=variables,
            extra_headers=extra_headers or None,
            queue_size=config.subscription_queue_size,
            timeout=float(config.timeout),
        ) as upstream:
            async for result in upstream:
                await ws.send_json({
                    "type": "next",
                    "id": sub_id,
                    "payload": {
                        "data": result.data,
                        "errors": result.errors or None,
                    },
                })

        # Stream complete
        await ws.send_json({"type": "complete", "id": sub_id})

    except WebSocketDisconnect:
        pass  # Client disconnected — clean up silently
    except ImportError:
        await ws.send_json({"type": "error", "payload": [{"message": "Install graphql-mcp[subscriptions]"}]})
        await ws.close()
    except Exception as exc:
        logger.exception("WebSocket subscription error: %s", exc)
        try:
            await ws.close(code=1011)
        except Exception:
            pass
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| downstream WS client → FastAPI WS endpoint | Untrusted WebSocket client sends graphql-transport-ws messages |
| FastAPI WS endpoint → upstream WS server | Brick proxies subscription to upstream, forwarding only allowed headers |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Spoofing | WS endpoint | mitigate | Forward only whitelisted headers (authorization, x-user-id, x-roles) from connection_init payload — same whitelist as REST (reuse `_FORWARDED_HEADERS`) |
| T-11-02 | Tampering | WS messages | mitigate | Validate message types (connection_init, subscribe) before processing; reject unexpected types with error response |
| T-11-03 | Denial of Service | WS endpoint | mitigate | Rate limiting middleware applies to WS upgrade requests (HTTP 429 before upgrade). Bounded async queue (default 128) prevents unbounded memory growth from slow consumers |
| T-11-04 | Information Disclosure | WS error messages | mitigate | Do not leak upstream connection details in error messages sent to downstream client; log full errors server-side only |
| T-11-05 | Elevation of Privilege | WS query injection | accept | Subscription query is forwarded as-is to upstream (same as POST /graphql/query pattern). Query depth checking is not applied to subscriptions (subscriptions are typically shallow). Upstream is the authority for query validation |
</threat_model>
