# Plan 13-01: Copier Template Extraction

## Objective

Create a Copier-compatible project template from the graphql-mcp skeleton so that `copier copy` generates a fully parameterized MCP brick with optional feature selection.

## Context

- @13-CONTEXT.md — Phase boundary and decisions
- @pyproject.toml — Build configuration to templatize
- @src/graphql_mcp/ — Source structure to templatize

## Tasks

<task id="1" type="auto">

### Task 1: Create copier.yml and template directory structure

Create the `template/` directory with `copier.yml` defining all user-facing questions and the Jinja2-based project skeleton.

**Done criteria:** `copier.yml` exists with questions for module_name, package_name, env_prefix, and optional feature flags.
**Verify:** `cat template/copier.yml` shows all expected questions.

</task>

<task id="2" type="auto">

### Task 2: Create Jinja2-templatized project skeleton

Create all templatized files under `template/`:
- `pyproject.toml.jinja` — parameterized project name, module, extras
- `src/{{module_name}}/` — core hexagonal architecture skeleton
- `Dockerfile.jinja` — parameterized CMD
- `tests/` — basic test skeleton
- `__init__.py`, config, domain models, ports, adapters

**Done criteria:** All template files exist and use Jinja2 variables consistently.
**Verify:** `find template/ -name '*.jinja' -o -name '*.py'` shows complete skeleton.

</task>

<task id="3" type="auto">

### Task 3: Create copier template test

Create `tests/test_copier_template.py` that:
1. Generates a project using copier.run_copy()
2. Verifies no hardcoded `graphql_mcp` or `graphql-mcp` strings remain
3. Verifies generated file structure matches expectations
4. Runs pytest in the generated project

**Done criteria:** Test file exists and can be executed.
**Verify:** `python -m pytest tests/test_copier_template.py -x -v` passes.

</task>

<task id="4" type="auto">

### Task 4: Verify all existing tests still pass

Run the full test suite excluding benchmarks and integration tests to confirm no regressions.

**Done criteria:** All existing tests pass.
**Verify:** `python -m pytest tests/ -x --ignore=tests/benchmarks --ignore=tests/integration -q` passes.

</task>

## Verification

```bash
python -m pytest tests/test_copier_template.py -x -v
python -m pytest tests/ -x --ignore=tests/benchmarks --ignore=tests/integration -q
```

## Success Criteria

1. `copier copy` with defaults generates a complete project structure
2. No hardcoded `graphql_mcp` or `graphql-mcp` in generated output
3. Generated project's basic test suite passes
4. All existing project tests continue to pass
