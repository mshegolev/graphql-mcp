# Plan 12-01: GitHub Actions Publish Workflow & SDK Examples

## Objective

Create a GitHub Actions workflow for PyPI OIDC Trusted Publishing triggered on `v*` tags, and an `examples/` directory with runnable SDK usage examples (basic query, async query, subscription).

## Context

- @pyproject.toml — maturin build system, existing wheel matrix in ci.yml
- @.github/workflows/ci.yml — existing CI with cibuildwheel, sdist, lint
- @src/graphql_mcp/__init__.py — exports GraphQLClient, AsyncGraphQLClient
- @src/graphql_mcp/adapters/inbound/lib.py — sync client API
- @src/graphql_mcp/adapters/inbound/async_lib.py — async client API with subscribe()

## Tasks

### Task 1: Create GitHub Actions publish workflow
type="auto"

Create `.github/workflows/publish.yml` that:
- Triggers on `v*` tag push
- Builds wheels via cibuildwheel (same matrix as ci.yml)
- Builds sdist
- Publishes to PyPI using OIDC Trusted Publishing (pypa/gh-action-pypi-publish)
- No API tokens needed — uses GitHub's OIDC identity

### Task 2: Create examples directory with SDK usage examples
type="auto"

Create:
- `examples/basic_query.py` — sync GraphQLClient usage
- `examples/async_query.py` — async GraphQLClient usage
- `examples/subscription.py` — async subscription streaming
- Each with inline docstrings and usage instructions

## Verification

- Workflow YAML is valid (parseable)
- Examples are syntactically valid Python
- Existing tests still pass

## Success Criteria

- `.github/workflows/publish.yml` exists and triggers on `v*` tag
- `examples/` contains 3 runnable example scripts with inline docs
