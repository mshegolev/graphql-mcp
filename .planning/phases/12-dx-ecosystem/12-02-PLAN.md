# Plan 12-02: Docker Compose Integration Harness

## Objective

Create a docker-compose.yml with a mock GraphQL server for local integration testing, and integration tests that exercise the full SDK against it.

## Context

- @pyproject.toml — project dependencies and extras
- @Dockerfile — existing production container build
- @src/graphql_mcp/adapters/inbound/lib.py — sync client to test
- @src/graphql_mcp/adapters/inbound/async_lib.py — async client to test

## Tasks

### Task 1: Create mock GraphQL server
type="auto"

Create `tests/integration/mock_server.py` — a minimal GraphQL server using only stdlib + graphql-core that:
- Serves a simple schema (Query { hello, users })
- Handles POST /graphql with JSON body
- Returns proper GraphQL JSON responses

### Task 2: Create docker-compose.yml
type="auto"

Create `docker-compose.yml` with:
- `mock-graphql` service running the mock server
- `integration` service that runs pytest tests/integration/
- Network connectivity between services

### Task 3: Create integration tests
type="auto"

Create `tests/integration/` with:
- `conftest.py` — fixtures with GRAPHQL_ENDPOINT pointing to mock server
- `test_integration.py` — tests exercising sync query, async query, schema resolution

## Verification

- Mock server script is valid Python
- docker-compose.yml is valid
- Integration tests pass when run against the mock server
- Existing unit tests still pass

## Success Criteria

- `docker compose up` starts mock GraphQL server
- `pytest tests/integration/` passes against it
- Tests cover sync query, async query, schema operations
