    **The `_entities` query construction:** Build the GraphQL query string that federation gateways expect:
    ```graphql
    query ($representations: [_Any!]!) {
      _entities(representations: $representations) {
        ... on Product { __typename }
      }
    }
    ```
    However, since this is a PASS-THROUGH (per CONTEXT.md decision), we do NOT need to parse `__typename` to build inline fragments. The simplest correct approach: construct a query with `_entities(representations: $representations) { __typename }` and pass the representations as the `$representations` variable. The gateway resolves the rest. The `__typename` field is always valid on any entity and ensures a non-empty selection set.

    The query string to send:
    ```python
    _ENTITIES_QUERY = """\
    query ($representations: [_Any!]!) {
      _entities(representations: $representations) {
        __typename
      }
    }"""
    ```

    Then call `self._transport.execute(_ENTITIES_QUERY, {"representations": representations})`.

    **In `lib.py` — add `entities()` method** after `raw()` (around line 156):
    ```python
    def entities(self, representations: list[dict[str, Any]]) -> QueryResult:
        """Resolve federation entities via _entities(representations:) pass-through.

        Sends the representations to the federation gateway's _entities query
        and returns the raw result. This is a query (not a mutation), so the
        mutation guard does not apply.
        """
        if self._transport is None:
            return QueryResult(
                errors=[{"message": "No endpoint configured"}],
                error_class=ErrorClass.TRANSPORT,
            )
        return self._transport.execute(
            _ENTITIES_QUERY,
            {"representations": representations},
        )
    ```

    Add `_ENTITIES_QUERY` as a module-level constant near the top of `lib.py` (after imports, before the class).
