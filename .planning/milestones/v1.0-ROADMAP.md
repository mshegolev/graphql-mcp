## Phase Details

### Phase 1: Foundation & Schema Sources

**Goal**: The hexagonal skeleton compiles and the schema cascade resolves a live or offline schema through all four source adapters.
**Mode:** mvp
**Depends on**: Nothing (first phase)
**Requirements**: GQL-06

**Success Criteria** (what must be TRUE):

1. `src/generic_graphql_mcp/domain/` imports no I/O or framework symbols — confirmed by `ruff` + a `grep -r 'import fastapi\|import requests\|import httpx' src/generic_graphql_mcp/domain/` returning empty.
2. A `GraphQLClient.from_env()` call in a bare Python script resolves a schema without raising, using whichever cascade source is reachable (GitLab token present → GitLab; else introspection; else `_service`; else local SDL file).
3. With introspection disabled and no GitLab token, the client silently falls back to `_service{sdl}` and then to an SDL file — verified by mocked adapters in a unit test.
4. The TTL cache returns the cached schema without a network call on repeated access within the TTL window.

### Phase 2: Operations, Errors & Federation

**Goal**: All 6 operations are callable through the lib facade and return typed results with correct error classification and federation metadata.
**Mode:** mvp
**Depends on**: Phase 1
**Requirements**: GQL-01, GQL-02, GQL-03, GQL-04, GQL-05, GQL-07, GQL-08, GQL-09

**Success Criteria** (what must be TRUE):

1. `client.query(query, variables)` returns `QueryResult{data, errors, error_class}` — `error_class="ok"` on success, `"graphql"` when the response contains `errors[]`, `"transport"` on HTTP != 200 or timeout — all three verified on a mock endpoint.
2. Sending a `mutation { ... }` string to `client.query()` or `client.raw()` raises `MutationGuardError`; the same call succeeds when `GRAPHQL_ALLOW_MUTATIONS=true`.
3. `client.introspect()` returns a `SchemaSummary` listing Query root fields; `client.describe_type("SomeType")` returns `TypeInfo{name, fields, args, subgraph}` with `subgraph` populated when a supergraph SDL is loaded and `null` when it is not.
4. `client.list_subgraphs()` returns a non-empty list of `Subgraph{name, url, owned_types}` parsed from a sample supergraph SDL; returns an empty list (not an error) when `GRAPHQL_SUPERGRAPH_SOURCE=off`.
5. `client.refresh_schema()` completes without error and forces the next schema access to re-fetch (verified by confirming the mock adapter is called exactly twice across two `introspect()` calls separated by `refresh_schema()`).

**Plans:** 2/2 plans complete

### Phase 3: Native & Faces

**Goal**: The pyo3 JsonCodec crate builds via maturin, the orjson fallback passes the same parity test, and all four inbound adapters (lib, mcp_stdio, FastAPI REST+MCP-over-HTTP, CLI) expose the full operation set.
**Mode:** mvp
**Depends on**: Phase 2
**Requirements**: GQL-10 (native + faces portion)

**Success Criteria** (what must be TRUE):

1. `maturin develop` in the repo root produces a working `generic_graphql_mcp_native` Python extension; `from generic_graphql_mcp.adapters.outbound.json_native import RustJsonCodec` imports without error.
2. The parity test suite passes: `RustJsonCodec` and `OrjsonCodec` return byte-identical output for parse/traverse/field-select/redaction on a 100 KB and a 1 MB JSON fixture.
3. `from generic_graphql_mcp import GraphQLClient` in a test with no running server or MCP process successfully calls `client.introspect()` against a mocked transport — library face works in-process.
4. `generic-graphql-mcp query '{ __typename }'` (CLI) exits 0 and prints JSON to stdout on a reachable endpoint.
5. `GET /health` on the FastAPI app returns `{"status": "ok"}`; `POST /graphql/query` with a valid query body returns a JSON response with `data`, `errors`, `error_class` fields.

**Plans:** 2/2 plans complete
